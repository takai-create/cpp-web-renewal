# お問い合わせフォーム GAS セットアップ手順

## 概要

Google Apps Script (GAS) を使ってフォーム送信の受信口を作成します。

**仕組み**:
1. サイトのフォーム → GASウェブアプリURLに POST
2. GAS がスプレッドシートに記録
3. GAS が管理者メール・自動返信メールを送信

**メリット**:
- 完全無料（Google アカウントのみ）
- 送信通数の制限なし
- スプレッドシートで一元管理
- 自動返信メールまで自動化

---

## セットアップ手順

### 1. スプレッドシート作成

1. https://sheets.google.com/ にアクセス
2. 「+」で新しいスプレッドシートを作成
3. ファイル名: `CPP お問い合わせ記録` など任意

### 2. GAS プロジェクト作成

1. スプレッドシートのメニューから **「拡張機能」→「Apps Script」**
2. 新しいタブで GAS エディタが開く
3. プロジェクト名を `CPP Contact Form` に変更

### 3. コードを貼り付け

1. デフォルトの `function myFunction() {}` をすべて削除
2. リポジトリの [gas/ContactForm.gs](../gas/ContactForm.gs) の内容を **全てコピーして貼り付け**

### 4. 設定を編集

コード上部の `CONFIG` オブジェクトを編集します:

```javascript
const CONFIG = {
  NOTIFY_EMAIL: "takai@crowdpower-partners.com",  // ← 通知を受け取るメール
  FROM_NAME: "Crowd Power Partners Website",
  SPREADSHEET_ID: "",  // ← 空のままでOK（コンテナバインド）
  SHEET_NAME: "お問い合わせ",
  SEND_AUTO_REPLY: true,  // ← 自動返信するなら true
  REPLY_FROM_NAME: "Crowd Power Partners",
  REPLY_SUBJECT: "【Crowd Power Partners】お問い合わせを受け付けました",
};
```

**変更が必要なのは `NOTIFY_EMAIL` のみ** です。

### 5. 保存

- Ctrl+S で保存
- 初回保存時にプロジェクト名の確認あり

### 6. 権限付与（初回のみ）

1. 関数一覧から `testSave` を選択
2. 「実行」ボタンをクリック
3. 権限承認ダイアログが出る:
   - 「権限を確認」
   - アカウント選択
   - 「詳細」→「(プロジェクト名)（安全ではないページ）に移動」
   - 「許可」
4. 実行完了したら、スプレッドシートに「お問い合わせ」シートが作成されテスト行が入る
5. 指定したメールアドレスにテスト通知メールが届く

### 7. ウェブアプリとしてデプロイ

1. 右上の **「デプロイ」→「新しいデプロイ」** をクリック
2. 歯車アイコン ⚙️ →「ウェブアプリ」を選択
3. 設定:
   - **説明**: `v1`（任意）
   - **次のユーザーとして実行**: 自分
   - **アクセスできるユーザー**: **全員** ← 重要
4. 「デプロイ」クリック
5. 権限承認を再度求められたら承認
6. 表示される **「ウェブアプリのURL」** をコピー
   - 形式: `https://script.google.com/macros/s/XXXXXXXXXX/exec`

### 8. サイトに設定

リポジトリの `data/content.json` を開き、`contactPage.formAction` に貼り付け:

```json
"contactPage": {
  ...
  "formAction": "https://script.google.com/macros/s/XXXXXXXXXX/exec",
  ...
}
```

コミット&プッシュ → 本番反映されます。

## 動作確認

### ローカルで確認

```bash
npm run dev
```

http://localhost:3001/contact を開いてフォームを送信してください。

**期待する動作**:
1. 「送信中...」→ 成功画面に切り替わる
2. スプレッドシートに1行追加される
3. 管理者メール（NOTIFY_EMAIL）に通知が届く
4. 送信者のメールに自動返信が届く

## トラブルシューティング

### エラー: 「送信に失敗しました」

**原因の可能性**:
- ウェブアプリURLが間違っている
- デプロイ時に「アクセスできるユーザー: 全員」にしていない
- CSP でGASドメインが許可されていない

**確認**:
1. ブラウザの開発者ツール → Network タブで POST リクエストを確認
2. レスポンスのHTTPステータスを確認
3. GAS エディタの「実行」タブでログを確認

### エラー: 「Too many invocations」

GAS の1日あたりの実行回数制限（6分/実行、90分/日）に達した可能性。
通常の利用では問題ないが、スパム攻撃時に発生しうる。

### 自動返信が届かない

- GAS の MailApp は1日100通/アカウントの制限あり
- 迷惑メールフォルダも確認
- `CONFIG.SEND_AUTO_REPLY` が true になっているか確認

## スプレッドシートの保存内容

| 列 | 項目 |
|----|------|
| A | 受信日時 |
| B | お名前 |
| C | 会社名・所属 |
| D | URL |
| E | メール |
| F | 電話 |
| G | 種別（日本語） |
| H | 種別コード |
| I | 本文 |
| J | 同意（○） |

---

## コードの更新方法

GAS のコードを更新した場合:

1. GAS エディタで編集 & 保存
2. **「デプロイ」→「デプロイを管理」**
3. 鉛筆アイコン（編集）をクリック
4. 「バージョン」→「新バージョン」
5. 「デプロイ」
6. **URLは変わりません**（再設定不要）

---

## セキュリティ

- ハニーポット（`website` 隠しフィールド）で自動bot対策済
- 必須項目バリデーションあり
- Google側でSpam対策も自動適用
- POST のみ受付（GETは動作確認レスポンス）

---

## 送信先の切り替え（通知メール変更）

`CONFIG.NOTIFY_EMAIL` を編集して再デプロイするだけ。

```javascript
NOTIFY_EMAIL: "new@example.com",
```

再デプロイ: 「デプロイ」→「デプロイを管理」→ バージョン更新
