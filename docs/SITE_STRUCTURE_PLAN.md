# サイト構成検討プラン

## 目的
コーポレートサイトのページ構成と主要コンテンツを整理し、開発着手前に必要な設計項目を明確にする。

## 現在の想定ページ
- ホーム
- 会社概要
- サービス
- 実績・事例
- ブログ/ニュース
- お問い合わせ
- プライバシーポリシー
- 利用規約

## 主要セクション
- ヒーロー/トップビジュアル
- 会社紹介・ビジョン
- サービス概要
- 実績・導入事例
- ニュース・更新情報
- お問い合わせCTA
- フッター（会社情報・リンク）

## サイト構成の検討項目
1. ナビゲーション構造
   - グローバルナビゲーションに含めるページ
   - モバイルメニューの階層
2. ホームページの主要タッチポイント
   - 何を最初に見せるか（ヒーロー／特徴／実績）
   - CTA の配置
3. サービスページ構成
   - サービス一覧 + 詳細ページ
   - サービスフロー／メリット／導入事例
4. 実績・事例ページの設計
   - 一覧と詳細、カテゴリー/フィルタ
   - 画像・紹介文・成果指標
5. ブログ/ニュース設計
   - 期間・カテゴリ・タグ
   - 検索/絞り込み（必要なら）
6. お問い合わせフォーム要件
   - 必須項目、確認ページ、送信完了ページ
   - 外部API/バックエンド連携の検討
7. SEO / メタ情報
   - ページごとの title/description
   - OGP・構造化データ
8. アクセシビリティ
   - キーボード操作
   - 色のコントラスト
   - フォームのラベルとエラーメッセージ

## 次に進めること
- `docs/PROJECT_PLAN.md` の Phase 2, Phase 3 を確認してページ設計と優先順位を決定
- `docs/STRUCTURE.md` を参照し、ディレクトリ構造とコンポーネント配置を最終調整
- 最初のページ構成案を作成し、以下を決める
  - ページタイトル一覧
  - 各ページの目的と主なコンテンツ
  - ナビゲーション構造

## 注意事項
- GitHub Pages の公開設定は現在後回しで問題ありません。
- まずはサイト構成を固め、開発に必要なページ・コンポーネントを洗い出すことを優先します。
