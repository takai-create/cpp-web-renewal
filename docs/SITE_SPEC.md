## 1. プロジェクト概要

### 1.1 目的
Crowd Power Partners株式会社のコーポレートサイトをフルリニューアルする。
約5年間更新されておらず、現在の事業実態と大きく乖離しているため、
最新の事業構成を正しく伝えるサイトに刷新する。

### 1.2 サイトの役割
- **「傘」としてのコーポレートサイト** — プロダクト紹介サイトではなく、思想で事業を束ねる
- 既存顧客・ビジネスパートナー・地域の方々・採用候補者に、最新の事業構成を正しく伝える
- 各サービスページからの流入に対し、会社全体の中での位置づけを誤解なく伝える
- 各サービスLP（Talk Insights等）は別設計。コーポレートサイトからリンクする構成

### 1.3 ターゲット（優先順）
1. 提携候補・既存顧客・ビジネスパートナー — 事業理解と信頼構築
2. 採用候補者 — カルチャーと成長機会の訴求
3. 地域の方々 — コワーキング等の地域事業の認知
4. メディア — ビジョンと実績の提示

### 1.4 コアメッセージ
> **「AIが動き、人が深める。」**

### 1.5 参考サイト
| 会社 | URL | 参考ポイント |
|------|-----|-------------|
| ラクスル | https://corp.raksul.com/ | 多角事業を統一ビジョンで束ねる構造、力強いコピー |
| LayerX | https://layerx.co.jp/ | 抽象ビジョンで多角化を正当化、カード型事業紹介 |
| Sansan | https://jp.corp-sansan.com/ | 1哲学で複数プロダクトを束ねる縦型展開 |

## 5. デザインシステム

### 5.1 カラーパレット

ブランドカラーはコーポレートロゴ（青 `#004ea2`）から逆算。

| 用途 | CSS変数 | カラーコード |
|------|--------|------------|
| ブランド（メイン） | `--v2-brand` | `#004ea2` |
| ブランド（暗） | `--v2-brand-dark` | `#003b7a` |
| ブランド（薄背景） | `--v2-brand-light` | `rgba(0,78,162,0.08)` |
| ダーク（見出し等） | `--v2-dark` | `#0a0a0a` |
| テキスト（本文） | `--v2-text` | `#333333` |
| テキスト（薄） | `--v2-text-light` | `#666666` |
| テキスト（淡） | `--v2-text-muted` | `#999999` |
| 背景（白） | `--v2-bg` | `#ffffff` |
| 背景（交互） | `--v2-bg-alt` | `#f7f8fa` |
| ボーダー | `--v2-border` | `#e8e8e8` |

### 5.2 タイポグラフィ

**フォント**: Noto Sans（英字）+ Noto Sans JP（日本語）
- フォールバック: Hiragino Kaku Gothic ProN → system-ui → sans-serif
- Google Fonts で読み込み（wght: 400, 500, 700）

| 要素 | サイズ PC / SP | ウェイト |
|------|---------------|---------|
| ヒーロー見出し | 56px / 32px | Bold (700) |
| セクション見出し | 36px / 24px | Bold (700) |
| サブ見出し | 20px / 16px | Medium (500) |
| 本文 | 16px / 14px | Regular (400) |
| ナビゲーション | 14px | Medium (500) |
| ボタン | 14px | Bold (700) |

### 5.3 ロゴ使用ルール

格納先: `public/images/logo/`

| 使用箇所 | ロゴタイプ | ファイル |
|---------|----------|---------|
| ヘッダー（白背景） | Aタイプ／カラー | 横長・シンボル左 |
| ヘッダー（ダーク背景） | Aタイプ／モノクロ（ネガ） | 横長・白 |
| フッター（ダーク背景） | Aタイプ／モノクロ（ネガ） | 横長・白 |
| ファビコン | シンボルマークのみ | Cタイプから切り出し |
| OGP画像 | Bタイプ／カラー | 縦型 |

元データ: `cpplogo2019_regulation_ol.ai`（Illustratorアウトライン化済み）
レギュレーション: `cpplogo2019_regulation.pdf`

### 5.4 レスポンシブ

| ブレークポイント | 幅 | 対応 |
|---------------|------|------|
| Desktop | 1280px〜 | フルレイアウト |
| Tablet | 768px〜1279px | 2カラム化 |
| Mobile | 〜767px | 1カラム、ハンバーガーメニュー |

**スマホメインの設計**: PCだけでなくスマホの表示を重視する。

### 5.5 アニメーション

| 要素 | 動き | タイミング |
|------|------|----------|
| セクション | フェードイン | スクロールイン（IntersectionObserver） |
| カード | 順次フェードイン | 0.2s間隔 |
| ヘッダー | 背景変化 + ぼかし | スクロール > 80px |
| ボタン | 背景色変化 | ホバー 0.15s ease |
| カード | 上昇2px + 影強化 | ホバー 0.2s ease |

**原則**: アニメーションは「気づかない程度」に。過度な演出は避ける。

## 11. 開発プロセス

### 11.1 制作フロー

```
デザイン作成 → オーナーレビュー → コーディング → 自動テスト → デプロイ
                                                  ↕
                                           セキュリティ検証
```

- デザイン先行: コーディング前にデザインでレビュー（トークン節約）
- 確認依頼時は必ずテスト環境URL or ドキュメントのありかを提示

### 11.2 テスト

**方針: 自動テスト（E2E + Lighthouse + リンクチェック）+ 手動テスト（オーナー確認）**

#### 自動テスト（CI自動実行）

| テスト | ツール | 検証内容 |
|--------|-------|---------|
| E2Eテスト | Playwright | 全ページ表示、ナビ遷移、フォームバリデーション・送信、SP表示 |
| SEO/性能 | Lighthouse CI | スコア70以上（パフォーマンス/SEO/アクセシビリティ） |
| リンクチェック | スクリプト | 全 `<a>` タグのリンク先が200を返すか |

#### 手動テスト（オーナー確認）

| 確認項目 | 確認方法 |
|---------|---------|
